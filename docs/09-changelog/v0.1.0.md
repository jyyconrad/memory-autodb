# v0.1.0 - 单 appId Working Context 闭环

## 新增

### Working Context 语义层（A1-lite）
- 新增 `core/scope-policy.ts`：多级 scope key（workspace/project/session）、`SlotReusePolicy` 复用策略、`applyScopeReusePolicy`/`applyVisibilityFilter`。单 appId 下 profile/rules 按 workspaceId 复用、task_context 按 projectId 隔离。
- `core/scope.ts` 新增 `validateScopeForWrite`：校验 tenantId/appId/userId 一致，防止跨用户错写（RISK-3）。
- `core/memory-service.ts` 支持可选 audit 注入：写入前 scope 校验，失败写 `scope.reject` 审计后拒写。
- `scopeToProjectKey` 修正包含 workspaceId，消除跨 workspace 同 projectId 碰撞。

### 快路径输出增强
- `ContextFastResponse` 新增 `filtered`/`filteredSummary`：12 类 `FilteredReason` 枚举解释被过滤记忆；warnings 规范化（`WarningCode` + `formatWarning`）。
- 新增 `core/recall-scoring.ts`：召回评分权重显式化（relevance/scopeFit/importance/confidence/evidenceWeight/recency），替代硬编码。

### 隐私安全
- 新增 `lifecycle/sensitive-filter.ts`：5 类敏感属性（人格/健康/政治/宗教/性取向）在 extractor 源头拦截，不进候选区、不进主库（RISK-15）。

### Project Memory Workspace（A2-lite）
- 新增 `adapters/openclaw/manifest.ts`：`.mengshu.json` schema + 稳定 id 生成（同目录幂等、移动后 identity 不变）。
- 新增 `ms init` + `ms project status/context/lookup` 命令。

### 本机接入体验（Milestone B）
- 新增 `ms doctor`：config/DB/embedding/model/磁盘/manifest 体检，区分 ok/warning/fatal（embedding 不可达为 warning 可降级）。
- 新增 `ms demo`：单 appId 样本工作上下文 + context/lookup 演示。
- 新增 `ms connect`：输出可复制的接入信息（URL/secret/scope/curl）。

### Console 候选区治理闭环（Milestone C）
- Console 新增 Candidates API（list/review/count）+ 前端 Candidates 页面 + Overview candidate backlog。
- `index.ts` 构造共享候选区（repository + review service），通过 `serve` 注入 daemon，Console 后端端到端可达。
- 新增 `lifecycle/candidate-promotion.ts`：approve 通过 `candidateToMemoryRecord` 写入主库。

### 记忆树自动构建（F3-1）
- 新增 `tree/build-tree-handler.ts`：`build_tree` job handler，leaf 追加到 buffer，满足 seal 条件时生成 SummaryNode。
- 扩展 `tree/seal.ts`：`sealBuffer` 支持可选 LlmClient，可用时生成 abstractive 摘要，失败降级到 extractive。
- 新增 `processing/llm-client.ts`：OpenAI 兼容 chat completion 客户端，支持重试、并发控制、NullLlmClient 降级。
- leaf 默认字段补全（entityIds=[]、importance=0.5、eventAt=now、tokenCount 按文本估算）。

### 评测基础设施（Milestone D-lite）
- 新增 `vitest.config.ts`（v8 coverage 80%/70%）+ `testing/fake-embeddings.ts`（1536 维 hash/constant 双模式）。
- 新增 `eval/`：v0.1 黄金集 30 条 + safety 黄金集 40 条 + quick-eval runner + LongMemEval 开源集 adapter。
- `npm run eval:quick` 脚本，release gate 失败退出码非 0，可接入 CI。

### observe 自动抽取链路接线
- `server/workers.ts` 新增 `startJobWorkerLoop`：daemon 在 listen 期间轮询 drain job 队列，stop 时清理；每 tick 用 seen set 防永久失败 job 饿死后续 job。
- `lifecycle/extract-candidate-handler.ts`：`extract_candidate` job handler，observe 文本经 extractor 抽取后写入候选区 pending（不污染主库）。
- `index.ts` 构造 `AgentFastPathService`（enqueueJob → ingestionStore.jobs），`serve` 注入 worker（`extract_candidate` handler）。
- `server/daemon.ts` `StartMemoryServerOptions` 新增 `agentFastPath` / `worker` 选项。
- 端到端验证：`server/observe-pipeline.e2e.test.ts`，observe → candidate 出现 < 100ms。
