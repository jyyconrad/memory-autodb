# v0.1.1 - MCP 真实接入 / 记忆树自动构建接线 / LLM 客户端

> 在 [v0.1.0](./v0.1.0.md) 单 appId Working Context 闭环基础上，补齐"配置好模型即可用"的三项能力。

## 新增

### MCP 真实接入（F1）
- 新增依赖 `@modelcontextprotocol/sdk@^1.29.0`。
- 新增 `adapters/mcp/stdio-server.ts`：用 `Server` + `StdioServerTransport` 绑定现有工具注册表，实现 `ListToolsRequestSchema` / `CallToolRequestSchema` handler，工具结果包成 MCP `content`。
- 扩展 `adapters/mcp/tools.ts`：8 个基础工具补 JSON Schema `inputSchema`；可选注入 `agentFastPath` 时额外暴露 `memory_context_fast` / `memory_observe_light` / `memory_lookup`（共 11 工具，不注入时保持 8 个向后兼容）。
- 新增 `ltm mcp` 命令（`adapters/openclaw/cli-mcp.ts`）：启动 stdio MCP server，SIGINT/SIGTERM 优雅关闭，状态信息走 stderr 不污染 JSON-RPC 流。
- `index.ts` 注入 `service` + `agentFastPath` 到 `ltm mcp`。
- 接入文档：[CLI 命令](../05-api/cli-commands.md) 含工具清单与 Claude Desktop 配置示例。

### LLM 配置 + 客户端（F2）
- `config.ts` MemoryConfig 新增可选 `llm` 块（provider/model/baseURL/apiKey/maxTokens/temperature）+ schema 校验 + allowedKeys。
- 新增 `processing/llm-client.ts`：
  - `OpenAiLlmClient`：OpenAI 兼容 chat completion，`complete` / `summarize`，p-limit 并发 + p-retry 重试。
  - `NullLlmClient`：未配置时的降级占位，`available=false`，调用即抛错（强制上层先查 `available`）。
  - `createLlmClient(config.llm)`：有配置返回 OpenAI 实现，否则返回 Null 实现。
- `config.example.json` 加 llm 块示例。

### 记忆树自动构建接线（F3）
- 新增 `tree/build-tree-handler.ts`：`build_tree` job handler，leaf 追加到 buffer，满足 seal 条件时调用 `sealBuffer` 生成 SummaryNode；payload 缺字段补默认值。
- 扩展 `tree/seal.ts`：`sealBuffer` 支持可选 `LlmClient`，可用时生成 abstractive 摘要（`metadata.summaryMode="abstractive"`），失败或不可用降级 extractive。
- `api/agent-fast-path.ts`：
  - F3-2：`observeLight` 在入队 `extract_candidate` 的同时入队 `build_tree`（source 树，按 sessionId 分组）。
  - F3-3：`lookup` `mode=deep` 时融合 `TreeSummaryNode`（source/topic/global），`fast` 模式退化为纯向量召回，保护 context_fast 快路径 SLO。
- `index.ts`：构造 `treeRepository`（in-memory）+ `llmClient` + `buildTreeHandler`，worker 注册 `build_tree`，`AgentFastPathService` 注入 `loadTreeSummaries`。

## 已知缺口 / 待人工验证

- **MCP 端到端未人工冒烟**：stdio handler 用 FakeTransport 单测覆盖，未用真实 Claude Desktop / Cursor 连接实测。
- **LLM 真实调用未验证**：LlmClient 用 fake client 单测，未真连 OpenAI chat API。配置真实 key 后需冒烟 `summarize`。
- **记忆树仍 in-memory**：v0.x 不持久化（与产品计划 §4.2 一致），重启丢失，持久化留 v1.x。
- **树构建仅 observe 触发**：当前只 observe → source 树；ingest pipeline 的 chunk 触发、topic/global 树构建策略（需 entity 抽取 / daily digest 调度）延后。

## 兼容性

- MCP `agentFastPath` 工具、LLM `llm` 配置块、daemon `agentFastPath`/`worker` 选项均为可选，未注入/未配置时行为与之前完全一致。
- 现有 OpenClaw 工具、CLI 命令、REST API、scope 字段全部兼容。
